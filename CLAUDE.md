   - Docker (optional, for containerised dev)
2. **Clone the repo**
   ```bash
   git clone <repo‑url>
   cd "C:\Users\Manish\OneDrive\Desktop\Web App"
   ```
3. **Install dependencies**
   ```bash
   # Frontend
   cd client   # or the appropriate folder
   npm ci   # or `yarn install`

   # Backend (if separate)
   cd ../server
   pip install -r requirements.txt   # for Python backends
   npm ci   # if backend also uses Node.js
   ```
4. **Environment variables**
   - Copy `.env.example` to `.env` and fill in the required values (DB connection strings, API keys, etc.).
   - For local development you can use the provided `dev.env` file.
5. **Run the app**
   ```bash

   npm run dev   # Starts both client & server (via concurrently) or use separate scripts listed below.
   ```

## 📦 Build & Deployment
1. **Production Build**
   ```bash
   npm run build
   ```
   - Frontend assets bundled into `build/` (React) or `dist/` (Vue).
   - Backend compiled/transpiled if necessary.
2. **Docker** (recommended for deployment)
   ```Dockerfile
   # Example Dockerfile (adjust paths as needed)
   FROM node:20-alpine AS builder
   WORKDIR /app
   COPY . .
   RUN npm ci && npm run build

   FROM nginx:alpine
   COPY --from=builder /app/build /usr/share/nginx/html
   EXPOSE 80
   CMD ["nginx", "-g", "daemon off;"]
   ```
   Build and push:
   ```bash
   docker build -t your‑repo/web‑app:latest .
   docker push your‑repo/web‑app:latest
   ```
3. **CI/CD**
   - GitHub Actions workflow `ci.yml` runs lint, test, build, and Docker image publishing on `main`.
   - Pull‑request checks enforce code‑review quality via Claude Code (`/code-review` skill).

## 🐞 Debugging & Troubleshooting
- **Common startup errors**
  - `Module not found` → ensure you ran `npm ci` in the correct directory.
  - Port conflicts → change `PORT` in `.env`.
- **Database connection**
  - Verify connection string, run `docker compose up db` if using a local container.
- **Performance**
  - Use Chrome DevTools for frontend bottlenecks.
  - Profile backend with `node --inspect` or Python’s `cProfile`.
- **Logs**
  - Frontend: console output in the browser.
  - Backend: stdout logs, or `docker logs <container>` when containerised.
